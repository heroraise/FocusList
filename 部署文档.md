# 专注清单 FocusList —— 部署与运行文档

本文档面向首次运行本项目的同学，按步骤操作即可把工程跑起来，并演示各项功能。

---

## 一、环境要求

| 项目 | 要求 | 说明 |
| --- | --- | --- |
| 操作系统 | Windows 10/11、macOS 或 Linux | 任意其一 |
| Android Studio | Koala (2024.1) 或更高版本 | 推荐使用自带 JBR |
| JDK | JDK 17 | AGP 8.5.2 不能使用 JDK 8 |
| Android SDK | Android 14 / API 34 | `compileSdk=34` |
| 最低运行系统 | Android 8.0 / API 26 及以上 | `minSdk=26` |
| 网络 | 可选 | “每日一言”需联网；离线时自动用内置句子兜底 |

版本信息：

| 工具 | 版本 |
| --- | --- |
| Gradle Wrapper | 8.7 |
| Android Gradle Plugin | 8.5.2 |
| Kotlin | 2.0.20 |
| KSP | 2.0.20-1.0.25 |
| Material Components | 1.12.0 |
| Room | 2.6.1 |
| Retrofit | 2.11.0 |

## 二、获取代码

从 GitHub 克隆：

```bash
git clone https://github.com/lgydhei/FocusList.git
cd FocusList
```

如果使用压缩包，解压后请打开包含 `settings.gradle` 的项目根目录。

---

## 三、导入并运行（推荐：Android Studio）

1. 启动 Android Studio。
2. 选择 `File ▸ Open...`。
3. 打开 `FocusList` 项目根目录。
4. 等待 Gradle Sync 完成。
5. 准备一台设备：
   - 模拟器：Device Manager 创建 API 26+ 设备，建议 API 34。
   - 真机：开启开发者选项和 USB 调试，并授权连接。
6. 点击工具栏绿色 Run 按钮运行 `app`。

如提示缺少 SDK，按 Android Studio 的提示安装缺失组件即可。

## 四、命令行构建

工程已附带 Gradle Wrapper。

```bash
# Windows
gradlew.bat assembleDebug

# macOS / Linux
./gradlew assembleDebug
```

生成的 Debug APK 位于：

```text
app/build/outputs/apk/debug/app-debug.apk
```

安装到设备：

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

如果命令行报错提示 Java 版本不兼容，请确认 `JAVA_HOME` 指向 JDK 17。Windows 上也可以临时使用 Android Studio 自带 JBR，例如：

```powershell
$env:JAVA_HOME="D:\AndroidStudio\jbr"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
.\gradlew.bat assembleDebug
```

## 五、首次启动须知

- Android 13+ 首次启动会申请通知权限，请点击“允许”，否则任务提醒与番茄钟常驻通知无法显示。
- 数据全部保存在设备本地：Room 数据库 + SharedPreferences。
- 卸载 App 会清除本地数据。
- 每日一言需要联网；离线或接口异常时会显示内置兜底句子。

## 六、功能演示路线

### 1. 待办管理

- 点击右下角 `+` 进入添加任务页。
- 输入标题、备注，选择分类、优先级。
- 打开“设置到期提醒”，选择日期和时间。
- 保存后返回列表，任务卡片左侧色条表示优先级。
- 勾选任务可标记完成；滑动任务可删除并撤销；点击任务卡片可编辑。

### 2. 任务提醒

- 创建一个 1-2 分钟后提醒的任务。
- 到点后，`AlarmManager` 触发 `ReminderReceiver`，系统弹出任务提醒通知。
- 设备重启后，`BootReceiver` 会重新排程未完成任务的提醒。

### 3. 番茄钟专注

- 切到“专注”页。
- 选择时长，选择关联任务或自由专注。
- 点击“开始专注”。
- 计时开始后会启动前台服务，退到桌面或锁屏后仍会继续计时。
- 通知栏提供暂停、继续、结束操作。
- 完成后会记录专注时长，并按设置播放提示音、振动或白噪音。

### 4. 数据统计

- 切到“统计”页。
- 查看今日专注分钟数、今日番茄、累计专注分钟数、累计番茄、待办任务数量。
- 最近记录列表默认只显示最近 7 天记录，最多 30 条；历史记录仍保存在数据库中，累计统计不会丢失。

### 5. ContentProvider 演示

- 在统计页点击“通过内容提供器查询任务”。
- App 通过 `ContentResolver` 调用 `TaskProvider`，弹出任务总数、待办数和已完成数。

### 6. 桌面小组件

- 回到桌面，长按空白区域。
- 选择 Widgets / 小组件。
- 找到“专注清单”并拖到桌面。
- 小组件会显示待办和已完成数量；点击刷新按钮可重新读取数据。

### 7. 设置页

- 点击右上角设置图标。
- 可调整默认专注时长、提示音、白噪音、振动。
- 设置项使用 SharedPreferences 保存，重启 App 后仍保留。

### 8. 横竖屏适配

- 专注页横屏：左侧显示每日一言与设置区域，右侧显示计时圆环。
- 统计页横屏：左侧固定显示统计面板，右侧记录列表可滚动查看。
- 旋转设备后会保留当前所在 tab，不会自动跳回待办页。

## 七、常见问题

**Q1：Gradle Sync 卡住或失败？**
检查网络；首次需要下载依赖。可点击 Android Studio 的 `Sync Project with Gradle Files` 重试。

**Q2：提示 “SDK location not found”？**
在工程根目录新建 `local.properties`，写入 SDK 路径，例如：

```properties
sdk.dir=C\:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk
```

Android Studio 正常打开工程时通常会自动生成该文件。

**Q3：命令行构建提示 Java 版本不兼容？**
请使用 JDK 17，不要使用 JDK 8。Android Studio 自带的 JBR 一般可直接使用。

**Q4：收不到提醒通知 / 番茄钟没有常驻通知？**
确认首次启动时已允许通知权限；也可在系统设置 → 应用 → 专注清单 → 通知中手动开启。

**Q5：每日一言一直是同一句？**
可能处于离线状态或接口限流，属于正常兜底；联网后点击刷新图标即可重新请求。

**Q6：模拟器没有小组件入口？**
回到 Launcher 桌面，长按空白区域选择 Widgets。部分模拟器桌面入口不同，需要先退出 App。
