# 专注清单 FocusList —— 项目说明

> 移动应用系统综合实践 · 北京信息科技大学
> 一款「待办清单 + 番茄钟专注」Android 应用，使用 Kotlin 实现，覆盖 Android 四大组件与课程要求的主要知识点。

---

## 一、项目简介

**专注清单 FocusList** 把“任务管理”和“番茄工作法”结合在一起：用户先把要做的事记成待办，再针对某个任务开启番茄钟专注计时；专注结束后自动统计时长，帮助用户量化和坚持自己的专注习惯。

当前 `v1` 版本在基础功能上补齐了更完整的界面体验：Material 3 主题、明暗色适配、横屏专用布局、统计页最近 7 天记录过滤，以及专注页更紧凑的计时与设置区域。

选择这一选题，是因为它能让 Android **四大组件自然且必要地协同工作**：

- 番茄钟必须在后台持续计时 → **前台服务**；
- 任务需要到点提醒、设备重启后提醒不丢 → **广播接收器**；
- 数据要能被桌面小组件读取 → **内容提供器**；
- 多页面交互 → **多 Activity / Fragment**。

## 二、核心功能

| 功能 | 说明 |
| --- | --- |
| 待办管理 | 新增、编辑、删除、完成状态、分类、优先级、提醒时间 |
| 番茄钟专注 | 选择时长、关联任务、开始/暂停/继续/结束，前台服务后台计时 |
| 专注统计 | 今日/累计分钟数、今日/累计番茄数、待办数量、最近 7 天记录 |
| 到点提醒 | AlarmManager 定时触发 BroadcastReceiver 弹出通知 |
| 桌面小组件 | 通过 ContentProvider 读取待办/已完成数量 |
| 每日一言 | Retrofit 获取一言接口，离线时使用内置兜底文案 |
| 设置项 | 默认专注时长、完成提示音、白噪音、振动 |
| 横竖屏适配 | 专注页和统计页提供 `layout-land` 横屏分栏布局 |

## 四、整体架构

```text
┌──────────────────────────────────────────────────────────┐
│                         UI 层                             │
│  MainActivity(BottomNav) ─ TaskList / Focus / Stats 三大 Fragment │
│  AddEditTaskActivity   SettingsActivity                   │
│            │ 观察 LiveData / 调用                          │
│        ViewModel(Task / Focus / Stats)                    │
└───────────────────────────┬──────────────────────────────┘
                            │
                    ┌───────▼────────┐
                    │   Repository    │  单一数据源
                    └───┬────────┬────┘
            ┌───────────▼──┐  ┌──▼─────────────┐
            │ Room(DAO/DB) │  │ Retrofit(网络) │
            └──────┬───────┘  └────────────────┘
                  │ 同一个 SQLite
        ┌──────────▼───────────┐
        │   ContentProvider     │◀── 桌面小组件 / 统计页 通过 ContentResolver 读取
        └──────────────────────┘

  Service(前台番茄钟) ── 计时/多媒体 ──▶ 通知；状态用 LiveData 回传 UI
  BroadcastReceiver(提醒/开机) ── AlarmManager 触发 ──▶ 通知
```

## 五、Android 四大组件落地

| 四大组件 | 本项目实现 | 文件 | 作用 |
| --- | --- | --- | --- |
| Activity | MainActivity、AddEditTaskActivity、SettingsActivity（+ 三个 Fragment） | `ui/**` | 承载待办、专注、统计、增改、设置等界面与交互 |
| Service | FocusTimerService（前台服务） | `service/FocusTimerService.kt` | 番茄钟倒计时在后台持续运行，并循环播放白噪音 |
| BroadcastReceiver | ReminderReceiver、BootReceiver | `receiver/**` | 接收 AlarmManager 定时广播弹出提醒；开机后重新排程 |
| ContentProvider | TaskProvider | `provider/TaskProvider.kt` | 以标准 Uri 暴露任务数据，供桌面小组件读取 |

组件协作示例：在待办列表点“专注” → 启动 `FocusTimerService` 计时；设置提醒 → `ReminderReceiver` 到点通知；桌面小组件 → 通过 `TaskProvider` 读取任务数量。

## 七、数据模型

**tasks（任务表）**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Long | 主键自增 |
| title | String | 标题 |
| notes | String | 备注 |
| category | String | 分类（学习/工作/生活/其他） |
| priority | Int | 优先级 0/1/2 |
| isDone | Boolean | 是否完成 |
| dueTime | Long? | 提醒时间戳，可空 |
| createdAt | Long | 创建时间 |
| pomodoroCount | Int | 累计番茄数 |

**focus_sessions（专注记录表）**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Long | 主键自增 |
| taskId | Long? | 关联任务，可空 |
| taskTitle | String | 任务标题快照 |
| durationMinutes | Int | 专注时长（分钟） |
| startTime | Long | 开始时间戳 |
| endTime | Long | 结束时间戳 |
| completed | Boolean | 是否完整完成 |

统计页列表使用 `endTime` 倒序展示最近 7 天记录，最多显示 30 条；历史记录不会自动删除，累计统计仍基于完整数据库计算。

## 八、主要模块说明

- **data/**：`Task`、`FocusSession` 实体；`TaskDao`、`FocusSessionDao`；`AppDatabase` 单例；`Repository` 统一数据入口。
- **network/**：`QuoteApi` 接口、`QuoteResponse` 模型、`RetrofitClient` 单例。
- **provider/**：`TaskProvider` 复用 Room 的 SQLite，支持 query/insert/update/delete 与变化通知。
- **service/**：`FocusTimerService` 前台计时、记录会话、多媒体；`FocusUiState` 通过 LiveData 把状态回传界面。
- **receiver/**：`ReminderReceiver` 弹出提醒；`BootReceiver` 开机后 `goAsync()` 重排闹钟。
- **widget/**：`TaskWidgetProvider` 通过 `ContentResolver` 查询任务数量并刷新桌面小组件。
- **ui/**：`MainActivity` + 三个 Fragment（待办/专注/统计）+ 各自 ViewModel/Adapter，`AddEditTaskActivity`、`SettingsActivity`。
- **util/**：`NotificationHelper`、`AlarmScheduler`、`SoundManager`、`TimeUtils`、`Prefs`。
